How it works

From changed code to a review your team can defend.

Hubolt keeps the model inside a bounded review workflow: gather context, check machine signals, generate findings, and record the outcome.

01 / Intake

Build the review packet.

Hubolt expands the diff with related files, semantic regions, ownership hints, and repository rules.

02 / Evidence

Run machine checks first.

Analyzers produce grounded signals before the LLM writes a finding, keeping guesses out of the review.

03 / Review

Route the model by risk.

Budget gates, provider routing, and rule memories shape the prompt before a finding reaches a reviewer.

Write the audit trail.
